# 2026-03-05 - Initial Recruit Tracker MVP

## 오늘까지 구현한 것

- **프로젝트 구조**
  - `backend` : FastAPI + SQLite
  - `frontend` : Next.js(App Router) + TypeScript
  - `updates` : 변경 내역 및 아이디어 기록용 폴더

- **데이터 모델 / 백엔드**
  - `Application` 엔티티
    - 날짜, 출처, 회사, 직무/포지션, 위치, 근무형태, 연봉/페이
    - 지원 여부, 단계(status), 공고 내용, 공고 URL, 메모
  - SQLite DB (`recruit_tracker.db`)에 영구 저장
    - 서버 재시작해도 데이터 유지
  - 엔드포인트
    - `GET /health`
    - `GET /applications`
    - `GET /applications/{id}`
    - `POST /applications`
    - `POST /applications/parse-from-url`
  - URL 파서
    - HTML `<title>` 기반으로 회사/포지션 후보 추출
    - 전체 HTML에서 텍스트를 정리해 공고 텍스트 후보 반환

- **프론트엔드 / UX**
  - Cloud Dancer 계열 파스텔 팔레트 적용
  - 상단 고정 헤더 + 2컬럼 레이아웃
    - 좌: 새 지원 추가 카드
    - 우: 지원 리스트 카드
  - 오늘 날짜 자동 입력 (수정 가능)
  - URL 입력 시:
    - 도메인 기반 출처(source) 자동 추론
    - `URL에서 자동 채우기`로 회사/직무/공고 텍스트 후보 설정
  - 공고 텍스트 붙여넣기 후:
    - `공고 텍스트로 자동 채우기`로 위치/근무형태/회사/직무 보정
  - 리스트
    - 테이블 형태
    - 단계(status)에 따라 뱃지 컬러 구분
    - 행 클릭 시 상세 페이지로 이동
  - 상세 페이지
    - 공고 텍스트를 섹션 키워드(예: `주요업무`, `자격요건`, `우대사항`, `복지`, `채용절차` 등)와
      문장 길이를 기준으로 분리해 여러 문단으로 렌더링

## 느낀 점 / 관찰

- 리멤버처럼 공고가 한 줄에 다 붙어 있는 경우에도,
  키워드와 마침표 기준으로 나누면 생각보다 꽤 읽기 좋게 정리된다.
- 출처, 위치, 근무형태 등은 완전 자동화까지는 어렵지만
  "적당히 맞는 값 + 사람이 한 번 정리" 정도만 되어도 입력 피로가 크게 줄어든다.
- Cloud Dancer + Ice Melt + Almost Aqua 조합이
  "채용 관리"라는 툴 성격과 잘 맞는, 부드럽고 부담 없는 인상을 준다.

## 앞으로 하고 싶은 것 (우선순위 초안)

1. **검색 / 필터 / 정렬**
   - 회사/포지션 텍스트 검색
   - 단계(status)별 필터 (관심/지원/OA/면접/오퍼/리젝트)
   - 최신/오래된 순 정렬

2. **리멤버 전용 파서**
   - `공고소개 / 주요업무 / 자격요건 / 우대사항 / 복지 / 채용절차 / 기타안내` 등
     섹션을 카드 단위로 나눠 보여주는 구조
   - 필요하면 사이트별(원티드, 사람인 등) 확장

3. **배포 준비**
   - SQLite → Postgres (클라우드 DB)
   - FastAPI : Render / Railway에 배포
   - Next.js : Vercel 배포, `NEXT_PUBLIC_API_BASE_URL` 환경변수로 백엔드 연결

4. **지원 단계별 통계/뷰**
   - 칸반 보드 (관심 → 지원 → OA → 면접 → 오퍼 → 리젝트)
   - 단계별 전환율, 회사/국가별 합격률 간단한 차트

5. **태스크 / 일정 탭**
   - 각 지원에 붙는 TODO(예: OA 대비, 면접 준비)와
   - OA/면접 날짜를 볼 수 있는 간단한 캘린더/타임라인
